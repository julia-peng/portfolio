Arrays

1D Arrays

Arrays provide an easy way to declare and share closely related data. Take the following example, intended to store the marks of every student in a class: 

int mark1 = 50;
int mark2 = 78;
int mark3 = 92;
int mark4 = 78;
int mark5 = 85;

As you can see, declaring an individual variable for each student is tedious and disorganized - the programmer would have to access each student's mark through its unique reference variable. Arrays provide an easy solution to this, allowing the programmer to group all of these data points under one name and location.

int[] allMarks = {50, 78, 92, 78, 85};

After storing the data under the shared reference variable allMarks, each data point can be accessed using its array index (allMarks[0] = 50, allMarks[1] = 78, etc.). This is a more organized way of storing the data. Additionally, another major benefit of arrays is ease of access. Since data points are accessed through data indexes, by using a for or while loop, the information can be accessed with high optimization.

//example code for printing each point in an array
for(int i = 0 ; i < array.length ; i++) {
    System.out.println(array[i]);
}

A common mistake made when using arrays is not being mindful of the arrays indexes. Remember - indexes start at 0, so that means the index of the last element in the arary will be array.length-1. This error comes out most often in automation, when the bounds of a loop are not properly adjusted to the array it is meant to iterate through.


More Arrays

2d Arrays

2d arrays can be thought of as an array of arrays. Let us consider the marking example again. Say that, instead of being created to represent one single classroom's grades, the programmer would like to create an array to represent all classes within a school. This can be done through the use of a 2D array. Using the principle of an 'array of arrays', an all-encompassing array can be created to contain the arrays containing each classroom's grades. Thus, the program has stored each student's marks in a 1D array representing a classroom. This classroom array is now stored in a 2D array representing the school as a whole. 

//declaration of 2D array holding a school's marks, organized into four classrooms
int[][] wlmacMarks = { {50, 78, 92, 78, 85},    //class 1
                     {93, 36, 75, 66, 89},      //class 2
                     {89, 45, 62, 59, 99},      //class 3
                     {94, 56, 83, 45, 87} }     //class 4

Parallel Arrays

Parallel arrays are separate arrays with corresponding data stored at the same data indices. For example, if there were two arrays all-encompassing one with names and one with marks, a parallel array would store one student's name at nameArr[2] and their mark at markArr[2].

String[] nameArr = {"Abraham", "Ben", "Charlie"};
int[] markArr = {100, 66, 34}; //the two different data types are stored at the same 
                               //index in each array

Why choose parallel arrays?

Indeed, corresponding data could be stored in 2D arrays. However, an advantage provided by parallel arrays is their ability to store and organize data that differ in data type, something that is not achievable through 2D arrays. For example, if one wanted to store a student's grade and name in a 2D array, it would not be very practical as the two differing data types (Strings and ints) would not be able to be stored in the same array without parsing. Thus, parallel arrays provide a useful tool here due to their ability to store corresponding data in two different arrays with differing data types.
