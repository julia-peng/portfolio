ArrayLists

ArrayLists are a form of upgraded array. Though arrays are an extremely valuable tool, they have some major limitations in Java, such as non-alterable length, difficulty adding data points and issues involving data storage. ArrayList objects provide a solution to these limitations. Using methods from the ArrayList class in the java.util package, data points can be added at any point, even beyond the ArrayList's original size.
ArrayLists can be declared as follows:

ArrayList<E> myList = new ArrayList<E>(); //'E' is used to represent a generic data type.
                                          //With this declaration, myList can store 
                                          //objects of any type.

//declaring an Integer ArrayList
ArrayList<Integer> intList = new ArrayList<Integer>(); //note that ArrayLists can only store
                                                       //objects, so the declaration must use 
                                                       //the equivalent wrapper class for ints 
                                                       //(Integers).

Let's revisit the unique capabilities of the ArrayList. Its most notable ability is the ability to 'insert' data at any given point with one simple method (add()).
Suppose the normal array myArr can be represented by the following table.

Index 0 1 2 3 4
Data 72 63 75 86 93

If the programmer were to want to insert a new data point into the array, bringing the total size up to 6, the steps would be quite long-winded. First, they would have to create a new array with a size of 6, insert all the data points up to the location of the new intended data point, add the new point and then insert the remaining data points from the new array. As you can see this is a long-winded process and also stores the new data in a new location.
ArrayLists eliminate such trouble. by using add(), the previously described process is fully executed through the built-in method, reducing hassle.
See the table below for a look into how add() works.

Index 0 1 2 3 4
Data 72 63 75 86 93 
myArr.add(2, 36);
Index 0 1 2 3 4 5
Data 72 63 36 75 86 93

A common mistake made when using ArrayLists is not being mindful of the storage type of ArrayLists - they store objects, not primitive data types. This means that, when accessing ArrayLists with Integers or Doubles, some of the operations that would have worked with these values in the form of primitive data types do not work in the context of the block-casted values in the ArrayList.
